# Activity Boundaries

## Authorized after activation

- read-only synchronization to the Sentinel-verified authorization-installation
  commit;
- canonical Revision 09 contract reads;
- source editing only in
  `pm_research/local_curl_per_side/prepared_evidence.py`;
- static AST/source/JSON/text/bytes/SHA-256 inspection without importing or
  executing project/authored modules;
- `git status --short --untracked-files=all`, Git diff, exact path inventory, and
  checksums;
- one R1 source-only checkpoint returned to Sentinel.

## Checkpoint form

The edited existing `prepared_evidence.py` is the only source file that may be
submitted. The static source report, exact path inventory, diff summary, and
checksums MUST be textual checkpoint content returned in chat. They MUST NOT be
created as new repository files. No implementation ZIP may be reconstructed.

## Unauthorized

Test-source editing; test collection, discovery, or execution; imports or
execution of project/authored modules; compilation, lint, type checking,
coverage, CI; another source-file edit; dependencies, CLI, config, generated
files, caches, bytecode; implementation ZIP reconstruction; research data or
empirical artifacts; API/RPC/vendor/Dune/curl/general network; Claude Git
commits, branches, pushes, PRs, merges, or remote writes; R2; P1/P2/P3; scoring;
probe execution; gate changes.
