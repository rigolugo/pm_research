## Per-request empirical states that are not whole-run stops

| Request-analysis condition | Compatibility outcome | Strict-audit treatment | Traversal/result effect |
|---|---|---|---|
| valid highest-attempt `INTERRUPTED` terminal | `INTERRUPTED_SELECTED`; `MEASUREMENT_INCONCLUSIVE` | `STRICT_NOT_EVALUATED_NONCOMPLETED`; counted in strict summary, no fabricated response row | continue; replay result permitted; included as unresolved in 496-token vector, 248 pairs, safeguards, bounds, and comparison |
| curl timeout/nonzero/runner exception after valid completed capture | `MEASUREMENT_INCONCLUSIVE` | strict front-end diagnostic when completed evidence exists | continue |
| absent/invalid/non-200 HTTP, including generic 404 | `MEASUREMENT_INCONCLUSIVE` | independent strict front-end diagnostic | continue |
| adapter eligibility unproven | C14 `MEASUREMENT_INCONCLUSIVE` | independent strict audit when completed bytes permit it | continue; never a confirmed transport transition |
| adapter eligible, current UTF-8 invalid | C15 `MEASUREMENT_INCONCLUSIVE` | strict UTF-8 result | continue |
| adapter eligible, UTF-8 valid, JSON invalid | C16 `MEASUREMENT_INCONCLUSIVE` | strict lexical JSON result | continue |
| compatibility valid but strict disagreement | compatibility state retained | `STRICT_COMPATIBILITY_DISAGREEMENT` | continue; run-level replay inconclusive |
| evidence oversize | `MEASUREMENT_INCONCLUSIVE` | quarantine; no response-dependent strict evaluation | continue |
| valid empty/no-window/present compatibility series | corresponding empirical terminal state | independent strict result | continue |

`NEVER_STARTED`, `STARTED_ONLY_UNRESOLVED`, and `COMPLETED_UNANALYZED` do not stop traversal immediately, but any such row at result closure triggers `STOP_REQUEST_POPULATION_INCOMPLETE`. A malformed interruption is never empirical evidence and triggers the applicable capture structural stop.

## Finalization distinction

- Successful structural stop finalization uses `FINALIZATION_PROFILE_STOP`, writes the stop inventory, and writes the marker last.
- Structurally valid replay finalization uses `FINALIZATION_PROFILE_REPLAY`, writes the replay inventory/result, and writes the marker last.
- `STOP_FINALIZATION_ATOMICITY_FAILURE` can leave a halted-but-unfinalized directory. The absent marker prevents any profile or result inference from partial files.

## Revision 23 response-header disposition

- Per-response `HEADER_STRUCTURE_INVALID`, present unsupported metadata, and unsupported absent-Content-Type body domains are C14 empirical inconclusive states, not whole-run stops. Recognized non-HTTP/1.1 selected protocols are the separate structural stop `STOP_HTTP_PROTOCOL_POLICY_VIOLATION`.
- A structurally valid selected-header status that disagrees with curl `--write-out` status is `STOP_ANALYSIS_CAPTURE_RECONCILIATION_INVALID`.
- A missing, hash-invalid, schema-invalid, or incompletely tested governing header contract/vector corpus is `STOP_RESPONSE_HEADER_EXTRACTION_CONTRACT_INVALID` before V7.
- Completed compatibility count plus valid `INTERRUPTED_SELECTED` count must equal 496. Started-only, never-started, and completed-unanalyzed remain closure stops.
- `STOP_FINALIZATION_ATOMICITY_FAILURE` may be halted but unfinalized; absent `FINALIZED.marker` means no finalized stop/replay profile and no replay label.

## Revision 23 lifecycle clarifications

The governing rows above are unique. Policy resolution requires exactly 32 registered fields and 68 registered observation rows. Valid `INTERRUPTED_SELECTED` is empirical unresolved evidence and is not a stop. Atomicity unsupported or atomic promotion failure leaves the run halted-unfinalized with no marker and no finalization profile. A post-finalization write is rejected externally and recorded only as `POST_FINALIZATION_WRITE_REJECTED` in the operational audit; it is not a typed run stop and does not mutate finalized bytes.
